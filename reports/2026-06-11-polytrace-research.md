## フレーム解析による発見

### 1. メッシュは時間的に「静止」がデフォルト【最重要】

`disguise.gif` の連続 3 フレーム(25fps)を比較した結果、**三角形分割は完全に同一**だった。
パラメータを触っていない間、メッシュは 1px も動かない。
「毎フレーム再抽出してパキパキ明滅する」のは Polytrace の挙動ではない。
GIF 内でメッシュが変化して見えるのは、作者が AE 上で設定値を変更している瞬間のみ。

### 2. ポイント配置は「ノイズエンジンで歪ませた格子」+「マスク点」

`noiseEngine.gif`: 黒画面に**規則的なドット格子を滑らかなノイズ場で変位させた波状のポイントフィールド**が表示され、連続フレーム間で**流体のように連続的に流れる**(AE のフラクタルノイズの evolution に相当する滑らかな時間発展)。
エッジ検出・輝度検出でポイントを置いている形跡はない。配置は手続き的(procedural)。

`customPoints.gif`: 顔の輪郭・眼鏡・口にユーザーがマスク点を打ち、その点がそのまま三角形の頂点になる。「every mask point becomes a Polytrace point」の実演。ディテールが欲しい場所の密度は**ユーザーがマスクで足す**(自動の内容適応ではない)。

### 3. 「グリッチ」は離散ではなくトポロジーのポップ + スライス

`polytrace-glitch.gif`: 三角形がバラバラに飛び散るシーンは**存在しない**。
グリッチの構成要素は:
- 点数(または閾値)を少なくした状態で点位置をノイズで動かす → Delaunay の**接続関係が不連続に切り替わる(トポロジーポップ)**。点自体は滑らかに動くのに三角形の形が瞬間的に変わる、これがグリッチ感の正体
- **水平スキャンラインのスライス/ブロックずれ**(おそらく別売り Glitchify の併用)
- メッシュ被覆率の増減(全面 → 数枚の三角形クラスタ → 消失)

### 4. 色は「静止メッシュの下を映像が流れる」

塗りは三角形ごとのフラット塗り(フッテージ色)。メッシュが静止していても**下の映像が動けば各三角形の色は毎フレーム変わる**。これが「ローポリのステンドグラス越しに映像が動く」独特の質感を生む。ワイヤーフレームは任意。

## 現実装との差分

| 観点 | Polytrace 実物 | 現 /polytrace 実装 |
| --- | --- | --- |
| 時間挙動 | 静止メッシュ(パラメータを動かした時だけ変化) | 毎フレーム再抽出で常時明滅 |
| 点配置 | ノイズ変位格子(手続き的)+ マスク点 | Sobel エッジ上位 N 点(内容適応) |
| 動きの源 | ノイズ evolution の滑らかな時間発展 / 色の更新 | 点の毎フレーム置き直し + ハッシュジッタ |
| グリッチ | 点移動に伴うトポロジーポップ + スライス併用 | (rollback 済み: 三角形の random 離散) |
| ディテール制御 | ユーザーがマスク点を追加 | エッジ感度スライダー |

## 忠実再現に向けた示唆

1. 点を**決定的な格子 + セル毎ハッシュオフセット**で固定し、**時間発展する滑らかなノイズ**(value noise の evolution)で変位させる → メッシュが粘性流体のように流れ、evolution 速度 0 で完全静止
2. 色サンプリングは毎フレーム継続(カメラが静止メッシュの下を流れる)
3. グリッチは「点数を絞る + evolution を速める」で自然発生するトポロジーポップに任せる
4. マスク点の web 翻訳として「クリック/ドラッグで点をピン留め」が Polytrace の customPoints 相当になる
5. 現行のエッジ適応配置は Polytrace には無い独自要素 — 残すならオプション扱いが妥当
